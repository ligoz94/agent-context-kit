# Spec Update PR Template

Use when implementation reveals under-specified behavior. See [Spec Update Workflow](../values.md#spec-update-workflow).

---

## Problem

What is underspecified? Reference the spec path and section.

## Proposed Change

Exact diff summary — what wording is added, removed, or clarified.

## Rationale

Why is this required for correctness? What breaks without it?

## Risk

What behavior changes as a result of this spec update?
